Cryptography functionalities for mitm.

crypto.new_RSA()#

Generates an RSA pair.

This function is intended to be utilized with new_X509(). See function new_pair() to understand how to generate a valid RSA and X509 pair for SSL/TLS use.

Parameters

bits (int) ‚Äì Size of the RSA key. Defaults to 2048.

Return type

PKey

crypto.new_X509(state_or_province_name, locality, organization_name, organization_unit_name, common_name, serial_number, time_not_before, time_not_after)#

Generates a non-signed X509 certificate.

This function is intended to be utilized with new_RSA(). See function new_pair() to understand how to generate a valid RSA and X509 pair for SSL/TLS use.

Parameters

  • country_name (str) ‚Äì Country name code. Defaults to US.

  • state_or_province_name (str) ‚Äì State or province name. Defaults to New York.

  • locality (str) ‚Äì Locality name. Can be any. Defaults to New York.

  • organization_name (str) ‚Äì Name of the org generating the cert. Defaults to mitm.

  • organization_unit_name (str) ‚Äì Name of the subunit of the org. Defaults to mitm.

  • common_name (str) ‚Äì Server name protected by the SSL cert. Defaults to hostname.

  • serial_number (int) ‚Äì A unique serial number. Any number between 0 and 2^159-1. Defaults to random number.

  • time_not_before (int) ‚Äì Time since cert is valid. 0 means now. Defaults to 0.

  • time_not_after (int) ‚Äì Time when cert is no longer valid. Defaults to 5 years.

Return type

X509

crypto.new_pair(cert_path)#

Generates an RSA and self-signed X509 certificate for use with TLS/SSL.

The X509 certificate is self-signed and is not signed by any other certificate authority, containing default values for its fields.

Parameters

  • key_path (Optional[Path]) ‚Äì Optional path to save key.

  • cert_path (Optional[Path]) ‚Äì Optional path to save cert.

Returns

Key and certificate bytes ready to be saved.

tuple
